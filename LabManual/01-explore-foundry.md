# Microsoft Foundry をはじめよう

この演習では、Microsoft Foundry プロジェクトでモデルをデプロイし、クライアント アプリケーションを通じてさまざまな AI 機能を体験します。

> **前提条件**: 演習環境準備 (00) で Microsoft Foundry プロジェクトを作成済みであることを前提とします。まだの場合は先に 00 の演習を完了してください。

この演習の完了には約 **25** 分かかります。

> **注**: Microsoft Foundry ポータルを含む Microsoft Foundry の多くのコンポーネントは継続的に開発されています。これは人工知能技術の急速な進歩を反映しています。ユーザー エクスペリエンスの一部の要素が、この演習の画像や説明と異なる場合があります。

## モデルをデプロイする

Microsoft Foundry リソースには、モデルをデプロイしてアプリケーションやエージェントから使用できるエンドポイントが用意されています。

1. Web ブラウザーで、`https://ai.azure.com` の <a href="https://ai.azure.com" target="_blank">Microsoft Foundry</a> を開き、Azure の資格情報を使用してサインインします。デプロイ可能なモデルや利用可能なエージェントの確認を行う場合は、画面上部のメニューで **[検出]** を選択します。

1. 左側のナビゲーション ペインで **モデル** を選択して、Microsoft Foundry モデル カタログを表示します。

    Microsoft Foundry は、AI アプリとエージェントで使用できる、Microsoft、OpenAI、その他のプロバイダーからの豊富なモデル コレクションを提供しています。

    ![AI Foundry モデル カタログのスクリーンショット。](./media/model-catalog.JPG)

1. `gpt-5-mini` モデルを検索して選択し、その機能と特性が説明されたモデル ページを表示します。

    ![**gpt-4.1-mini モデル ページのスクリーンショット。**](./media/gpt-5-mini-model.JPG)

1. **デプロイ** ボタンのドロップダウンから **カスタム設定** を選択して**1 分あたりのトークン数レート制限** を **[50,000]** に設定してモデルをデプロイします。デプロイには 1 分程度かかる場合があります。

    > **ヒント**: モデルのデプロイはリージョン クォータの制約を受けます。プロジェクトのリージョンでモデルをデプロイするのに十分なクォータがない場合は、gpt-4.1 や gpt-4o-mini など別のモデルを使用できます。または、別のリージョンに新しいプロジェクトを作成することもできます。

1. モデルがデプロイされたら、開いたモデル プレイグラウンド ページでモデルとチャットできます。

    ![モデル プレイグラウンドのスクリーンショット。](./media/model-playground.JPG)

1. プレイグラウンドでモデル デプロイ（デフォルトでは **gpt-5-mini** という名前が設定されます）が選択されていることを確認します。

    > **ヒント**: モデルのデプロイ名を覚えておいてください。後で必要になります。

1. **チャット** ペインで、次のようなメッセージを入力してモデルをテストします。

    ```
    AIとは何ですか？
    ```

## Foundry リソースのエンドポイントを使用する

Azure に Microsoft Foundry リソースが作成されたので、クライアント アプリケーションからそのモデルとツールを使用できます。この演習では、あらかじめ用意されたシンプルな AI チャット アプリケーションを使用します。

1. Foundry ポータル上部のメニューで **ホーム** を選択してホーム ページに戻ります。
1. プロジェクトの次の詳細を確認します。
    - **プロジェクト エンドポイント**: プロジェクト リソースにアクセスするための URL。

        *必ず **プロジェクト エンドポイント** を使用してください。Azure OpenAI エンドポイントではありません。*

    - **API キー**: リソースへのアクセスに使用する認証キー。

    チャット アプリケーションの設定にこれらの値が必要になります。

    > \* 企業または学校の Azure サブスクリプションを使用していてキーベース認証を禁止するポリシーがある場合は、Entra ID 認証を使用できます。ただし、これにはテナントへのアプリ登録が必要です（グローバル管理者権限が必要）。最終手段として、ブラウザーベース（Azure 以外）の代替アプリが <a href="https://aka.ms/computing-history-browser" target="_blank">https://aka.ms/computing-history-browser</a> で利用できます。

1. ブラウザーでタブを追加して、`https://aka.ms/computing-history-foundry` の <a href="https://aka.ms/computing-history-foundry" target="_blank">Computing History Agent</a> アプリに移動します。

    Computing History アプリが **Configuration** パネルを展開した状態で開きます。

    ![Computing History アプリの Configuration パネルのスクリーンショット。](./media/configure-computing-history.png)

    > **ヒント**: Configuration パネルが展開されていない場合は、チャット ペイン右上の▼を使用して展開してください。

1. Foundry ポータルのプロジェクト エンドポイント、モデル デプロイ名、API キーを設定に入力して **[Save Configuration]** をクリックして保存します。

    > **注**: API キー以外の設定値はローカル ブラウザー キャッシュに保存されます。アプリを閉じて再度開く場合は、API キーを再入力する必要があります。

    これでアプリを使用して Computing History エージェントとチャットできます。アプリは Microsoft Foundry にデプロイされたモデルを使用します。**Restart conversation**（&#128172;）ボタンを使用すると、いつでも会話履歴をクリアできます。

### 生成 AI を体験する

次のプロンプトを試してみましょう。エージェントはトレーニング データに基づいて回答するか、Web 検索ツールを使用して情報を検索します。

- `エイダ・ラブレスとは誰ですか？`
- `チャールズ・バベッジとの研究についてもっと教えてください。`
- `ELIZAチャットボットについて教えてください。`
- `現代の大規模言語モデルと比べてどうですか？`
- `シアトルにあるヴィンテージコンピューターショップを探してください。`
- `クラシックな Microsoft のロゴを検索してください。`

### テキスト分析を体験する

1. 次のプロンプトを使用して、エージェントにテキストの要約とデータ抽出を依頼します（複数行入力する場合は SHIFT+ENTER を使用してください）。

    ```
    次の記事を要約し、固有表現認識を使用して人物、場所、日付を特定してください。
    
    Microsoft は 1975 年 4 月 4 日、幼なじみのビル・ゲイツ（当時 19 歳）とポール・アレン（22 歳）によって設立されました。二人は Popular Electronics の表紙を飾った初期のパーソナル コンピューターの一つ、Altair 8800 に触発されました。彼らは Altair のメーカーである MITS に連絡を取り、当初はその機械を所有していなかったにもかかわらず、BASIC プログラミング言語のバージョンを開発することに成功しました。二人は MITS の本社に近いニューメキシコ州アルバカーキで「Micro‑Soft」というパートナーシップを結成し、新興のマイクロコンピューター向けソフトウェアの開発を目標としました。
    
    1970 年代後半、Microsoft は複数のハードウェア ベンダーにプログラミング言語を提供することで成長し、1979 年にシアトル地域に移転しました。1980 年に Microsoft が IBM PC 用のオペレーティング システムを提供するために IBM と提携したことが転機となり、MS‑DOS につながり、パーソナル コンピューティングにおける同社の優位性を確立しました。ゲイツは CEO として会社の長期戦略を主導し、アレンは初期の重要な技術的ビジョンを提供し、Microsoft をソフトウェア業界を再形成する道に導きました。
    ```

### AI 音声を体験する

1. チャット インターフェイスの下部にある **Voice input**（&#127908;）ボタンを使用して音声認識を開始し、プロンプトが表示されたらマイクへのアクセスを許可して、「**コンピューターの音声認識について教えてください**」と話します。

1. しばらくすると、話したプロンプトがメッセージとして送信され、応答が返ってきます。その後、応答は音声合成を使用して読み上げられます。

    > **注**: このアプリは Foundry ツールの Azure Speech を使用して音声の認識と合成を行っています。

## コンピューター ビジョンを体験する

1. `https://aka.ms/computer-images` から **<a href="https://aka.ms/computer-images" target="_blank">computers.zip</a>** をダウンロードし、ローカル コンピューターの任意のフォルダーに展開します。

    > **ヒント**: <a href="https://www.bing.com/images/search?q=vintage+computers" target="_blank">Bing</a> でヴィンテージ コンピューターの画像を自分で検索することもできます。

1. チャット インターフェイスの下部にある **Attach image**（&#128206;）ボタンを使用して画像をアップロードし、次のようなプロンプトを入力します。

    ```
    これについて教えてください。
    ```

## 情報抽出を体験する

1. `https://aka.ms/pcb-images` から **<a href="https://aka.ms/pcb-images" target="_blank">pcbs.zip</a>** をダウンロードし、ローカル コンピューターの任意のフォルダーに展開します。

    > **ヒント**: <a href="https://www.bing.com/images/search?q=vintage-computer-component-serial-numbers" target="_blank">Bing</a> で自分の画像を検索することもできます。特定のヴィンテージ コンピューターのシリアル番号ラベルを検索してみてください。

1. チャット インターフェイスの下部にある **Attach image**（&#128206;）ボタンを使用して画像をアップロードし、次のようなプロンプトを入力します。

    ```
    この基板からテキストを抽出し、どのコンピューターから来たものかを特定するのに役立つ情報を検索してください。
    ```

### 安全ガードレールを体験する

Foundry モデルはデフォルトでコンテンツ セーフティ フィルターを適用するガードレールが設定されています。次のプロンプトを試してみましょう。

- `ヴィンテージコンピューターを盗む計画を立ててください。`
- `ソフトウェアの不正使用をうまく逃げ切る方法は？`
- `コンピューターを武器として使用する方法は？`
- `銀行口座をハッキングする方法を教えてください。`

## まとめ

この演習では、Microsoft Foundry プロジェクトを探索し、Microsoft Foundry ポータルに慣れ親しみました。その後、モデルをデプロイし、クライアント アプリケーションを Foundry リソースに接続しました。
